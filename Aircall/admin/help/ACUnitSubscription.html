AC Unit Subscription


• When any unit will be purchased from anywhere in the system using the recurring billing with Credit Card or using Check/PO from employee application, we are adding number of subscriptions in our system according to the Plan Duration.


• i.e. Plan duration is 3 years, then 36 invoices are added into our database with the details of Unit and Payment Method.


• From this section, Admin can see a listing of all invoices for particular unit.


• Listing of the invoices with status: “Paid” / “Unpaid” / “Failed” will be shown with the date of last successful payment was placed or due date for unpaid invoice.




• Admin can filter the list of invoice using Client Name, date range, status, and payment method. Once results will display, admin will have the option to enter check number, check mark invoice as paid, and submit to change status to paid. When page reloads, results will display with check number (if entered) and status as paid as displayed in the below image.


• For the invoices which has payment method Check/PO, To mark them as paid, Admin have to select all those invoices and will enter check no if any, and click on ‘Mark subscription as Paid’ button.



• If the payment method will not be changed from cc to check, then the payment will be done automatically from the system and the status will be changed to “Paid” for that invoice of unit with successful payment.


• For the recurring billing, we will run the scheduler daily to charge the payment from the credit card using Stripe.


• However, if the payment method is Check / P.O., then admin has to add the details of Check / P.O. and manually change the status to “Paid” when the Check / P.O. will be received from the client.



• A friendly reminder will be sent client before X days of due date using email. Admin can change that email template from email template section.


• If payment is not received till due date, then system will send 3 types of cancellation warning to the client:

               

               1) 1 day past due date

               2) 8 day past due date

               3) 14 day past due date


• These are three different types of email templates (editable in the email template section). Similar automated alerts will be sent via push notification manageable from admin.


• Admin can change the payment method for any unpaid invoices of a particular unit.




• To change the method, Click on edit button, Detail page will be opened as displayed in the below image.


• From that page, Admin can change payment method to Check/PO from CC and vice versa.


• After changing payment method of any one invoice, then for all other ‘Unpaid’ invoices for that unit, payment method will be changed.


• Only Admin can change the payment method from the Admin Panel. The client has to contact admin if he / she wants to change the payment method of the purchased unit.


•  Taking payment using check is available via employee application only. Client Application / Client Portal / Admin Panel will not have this functionality. By default, for these portals, the payment method will be Credit Card only.


• If the unit will be purchased using “Special Offer” (Full Payment), then only single entry for subscription will be added. For this type of special offer payment, admin cannot change the payment method as full payment was accepted.


• If a unit that was supposed to be paid by check or p.o was not paid by due date, other than sending alerts and emails as described above, the system doesn't take any action and services continue to be provided. In the extreme measure where client did not pay and he is not coming back, Admin has the ability to cancel his plan manually. Cancellation is permanent and to reinstate a plan, client must setup a new unit. Old data will be archived.
