  1. Inventory Management

Part List


• The list of the part with the part name , part type, inventory type( i.e., inventory or non-inventory), Total acquired quantity, In stock quantity, selling price, status (i.e., Active/Inactive) with edit option will be shown in the listing.


• Inactivated or deleted part will no longer assignable to the employee to do any unit services.


• Parts those are not assigned to any services before can be deleted by admin.




• Admin can add a new part of the part type for the inventory or non-inventory.


• The parts types are the same as the mentioned in the daily part list.


• If any part has low stock then an email will be sent to the admin & warehouse manager with the list of part items which needs to be ordered.


• In-stock Quantity - Reserved Quantity <= Re-order Quantity this condition should fulfill to get the low stock part information.


o The fields marked (*) are compulsory.



Field

Function

Inventory Type (*):

Type of part.

There are two types of part.

  1. Inventory
  2. Non inventory

Non inventory part is not commonly used in unit service.

Part Type:

The part type will be fixed.

Part type is optional to select.

Part type drop down will show only the parts belonging to Unit Type Edit options. i.e. Compressor, TXV Valve, Filter, Fuse etc.

For other parts like Gas Tube, Exhaust Hose etc. part type do not need to be selected.

Parts added by selecting a particular “Part Type”, will load Unit Type Add/ Edit options from this section.

Part Name (*):

Name of the part.

Part Size :

Size of the part.

Part Description:

Description of the part.

Inbound quantity:

This defines the amount of this part that is on order from the manufacturer, or supplier. This will let the admin know if that part is on order.

Receive Quantity:

When getting new inventory, admin will use this field to enter the received quantity.

Total Acquired Quantity (*):

Total quantity of the part purchased since part inception.

In-stock Quantity:

Remaining quantity of part in the warehouse.

Reserved Quantity:

Quantity of reservations made for this part (collected from employee part requests)

Purchased Price (*):

The cost to purchase the part.

Selling Price (*):

The selling price at which we will sell the part to the client.

Minimum re-order Quantity (*):

Minimum required quantity when ordering this part.

Re-order Quantity (*):

Minimum number under which new stock needs to be ordered. Alert admin.

Status:

Status of part.

It can be either Active or Inactive.

Is Default:

If it is true then it will be added in daily part list as a default part.If nothing mentioned then employee will take this part to do the service.


• Part Name will be taken as unique key while adding new Part. There is no relation of Part Type with the uniqueness of Part Name.


• Admin can not give the same Part Name as already added Part Name though it belongs to other Part Type.


• Means, i.e. One Part with Part Name: “A” & with Part Type: “Filter” is added into the system. Then we can not add another part with Part Name: “A” & Part Type: “Fuse”.
