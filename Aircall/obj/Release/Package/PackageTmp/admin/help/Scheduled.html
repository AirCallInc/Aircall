  1. Services

Scheduled


• These type of services will be in scheduled section:

1) Any units’ services or requested services are approved by the client.


2) The system directly schedules the service with without client’s approval.


3) Admin manually adds the new schedule for the service.

Note: Click will not be notified to approve the services of type emergency, repairing and continuing previous work. For these types of services, client will get the scheduled service notification only.


• Admin can add the new schedule from this section. Admin will assign the employee to the client for the specified date. And the client will get the notification that service is scheduled for him/her on this date.


                                       



• Admin can assign the employee only if his work area matches to the client’s work area.


• Admin will get the employee list according to the selected client’s work area.


• Admin can edit the schedule service if needed and can change the employee and assign another employee.


• Admin can reschedule the service and can also see the rescheduled log. If any services rescheduled by the client, admin or employee.


• Admin can delete the requested scheduled service. Only requested services will have check boxes to delete that services.




• Admin can search through the service case #, client name, employee name, work area, date range etc.


• Scheduled services will be displayed in the client application > upcoming schedule.


• The client can reschedule the service after X days according to the type of the service (i.e Maintenance, Repairing, Emergency, Continuing previous work).


• Admin can manage X days from the site settings section of the admin panel.


• Admin has to provide following information for adding a new request for clients:


               The fields marked (*) are compulsory.

Field

Function

Client Name (*):

Name of client for whom admin wants to add services.

From searchable list of client, admin has to select client and select address for which need to add request.

Based on selection of client and address, all the units located at selected address and phone numbers of client will populate (Mobile, Home & Office).

Service Unit (*)

Select the units, which needs to be serviced.

Can able to select multiple unit.

Service Request Date (*):        

The date when the client has requested to add new service.

Purpose of Visit (*):

Admin can request for below things:

  • Maintenance
  • Repairing
  • Emergency
  • Continuing Previous Work

Work Area (*):

Select work area from searchable group of work area.

Employee work areas can be managed from Employee Management section.

Technician (*):

Technician for unit service.

All the technicians which are assigned to selected work areas will be displayed here.

Service Scheduled On (*):

Date for unit service schedule.

Schedule Start Time (*):

Start time of service.

Schedule End Time (*):

End time of service.

Customer Complaints (*):

Previous service notes will be displayed here.

Dispatcher Notes:

Notes of dispatcher.

Technician Notes:

Notes of technician.

Daily Part List:

Part (material) required for service.

This is pre-populated and calculated from unit detail section.

The daily part list items will be displayed unit wise.

Service Report History:

Admin can view all the past service reports for client unit.


Late Rescheduling policy:

• When client try to reschedule the service from upcoming schedule before 24 hour of service time, client will show alert with message like “You are going to reschedule the service less then 24hr before the meeting. Maybe you will charge a late reschedule penalty. Are you sure you want to reschedule the service?”.


• Message can change from site setting.


• Alert will have 2 buttons ‘Yes’ and ‘No’, If client press yes button from alert dialog then service status will automatically change to no show, after that service can reschedule by admin.


• This type of no show service will be highlighted with red line in no show service listing section in admin panel. So by the colour, the admin can decide which are added by the client and which are added by the employee.


• Once service’s status set as no show then it will be handle as simple no show service.(Refer section of No Show Service)


• After press “Yes” button user will redirect to previous screen and it will show message like “Your service was rescheduled successfully. Further instructions will be sent promptly”, this message also can change from site setting in admin panel.
