Plan Management

  1. The eco system is dependent on service contract packages / plans, as such, the ability to add available Plans which are provided to the customers in the Admin Panel.


  1. There are four types of Plans available in the system:

    1. Residential Plan
    2. Commercial Plan
    3. Industrial Plan
    4. Multi-Family Plan


  1. See below screenshot showing the listing of the Plans from where Admin can add, edit, delete, activate & inactive plans:



  1. Admin can add Plan by using any of above four Plan Type. Only single plan can be added for single Plan type.


  1. There is a limitation of only four plans.


  1. A plan can only be deleted if that plan is not associated with any unit in the system.


  1. By clicking on “Add Plan” button, below screen will be displayed.



    1. Below are the fields to add new plan :

Field

Functions

Plan Name (*)

Name of the Plan

Plan Type (*)

Plan type of the plan.

There are different 4 types of plan.

  1. Residential
  2. Commercial
  3. Industrial
  4. Multi-family

Package A Display Name (*)

Name for Package A.

Package A is used for the units which are more than 10 years old.

Package A Description (*)

It includes all the information for the package A like what type of services provided etc.

Package B Display Name (*)

Name for Package B.

Package B is used for the units which are less than 10 years old.

Package B Description (*)

It includes all the information for the package B like what type of services provided etc.

Plan Price in $ (*)

Price for the plan per month.

Defined amount, which will be used for recurring payment.

Duration (*)

Duration of the plan in month.

Mandatory Services (*)

Number of services will be provided to the client when any client will purchase plan.

Schedule First Service In (*)

The client will receive their 1st schedule service within specified days.

Drive Time (*)

Time given to an employee to reach at the client's location by driving. (In Minutes)

Service Time for First Unit (*)

Time given to an employee to do the service of first unit. (In Minutes)

Service Time for Additional Units (*)

Time given to an employee to do the service of other unit. (In Minutes)

Show Special Price

If yes, then the client is able to view special pricing for a plan in client application / client portal.

Show Auto Renewals

Offer option to show auto renewal option.

Discount Price (*)

If Show Special Price is Yes, show special price. If no, do not offer special price. If yes, and the client wants to pay for the whole contract price at the single time instead of recurring payment then specified discount price will be charged to the client. (Duration/ i.e. 36 month).

Background Color (*)

Specified color will be displayed for the plan as background in PlanCoverage in client application & client portal.

Plan Image

Image for the plan is displayed in PlanCoverage in client application & client portal.

Service Slot 1

Morning time slot. Specified time slot will be displayed while Request of Service and Reschedule of service in client application & portal.

Service Slot 2

Noon time slot. Specified time slot will be displayed while Request of Service and Reschedule of service in client application & portal.

Status

Status of the plan.

Status is either Active or Inactive.


  1. To add a plan, Admin must add all fields marked mandatory.


  1. When a adding a unit using client, employee application or client portal all active plans will be displayed.


  1. Available plans in the Admin Panel are displayed in the Client & Employee Application and Client Portal



  1. Client Application                                                                Employee Application


                                                     


  1. If any of the plans will be inactivated by the Admin then inactivated plans can not be shown in the application or in the portal.


  1. All added Plans will be displayed in the Plan Coverage section with plan image and background color in the client application & client portal as shown in the screenshot.
