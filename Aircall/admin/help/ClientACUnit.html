Client AC Unit


• Admin can add, edit, delete units from this section.


• AC units added by the client, admin or employee are displayed in this section.


• The listing will show the Client name, plan type, unit name, status, address, and edit option to edit the unit detail.


    • The fields marked (*) are compulsory.



Field

Function

Client Name (*):

Name of client for whom admin wants to add ac unit.

Here a client is selected from a searchable list.

Address of Unit (*):

Address where the unit is located.

Based on selection of client, all addresses of selected client are displayed.and it not added yet then option to add the new.

Each plan has two packages

  1. Package A (Unit < 10 years)
  2. Package B (Unit > 10 years)

  1. Packaged
  1. Split (if this is chosen, double all entry fields)

Unit Status:

Defines the status of the AC unit.

The Status of AC unit is either Serviced (green), Service Soon (orange), or Need Repair (red).

When a new AC unit was added at that time status of each unit is Service Soon (orange).

Technician will go to that unit located address and check that all fields are perfect or not.

If everything is perfect, then admin will update status to Serviced (green) and proceed to schedule the first appointment.



• Units can also be added from the client application and the client portal.


• If the plan to which unit will be added, is inactivated by admin then that plan will not be shown in the client/ employee application during adding the new unit.


• If the admin cancel the subscription for the plan then unit attached to that plan will be removed from this listing and from the client application’s my units section but it will remain in the database.


• After adding the units, X services of that units decided by the admin will provide the user within X months of intervals.



       


                                       

               


• If any unit is in the cart or payment is failed for the unit then when client/employee going to add new unit he/she will be prompted that the old un-purchased units are there you want to purchase them or not.


• If client wants to change the address after adding units then following cases may possible:


Case 1: The unit is added but the scheduler is not run yet. So the services will not be added.


• In this case, the Admin can change the address of that added by clicking on the edit button from the listing .


Case 2 :The unit is added and also the scheduler is run and the services will be added.


•In this case,

Suppose the client has added three units : AC1, AC2, AC3 on the Address 1


So the services will be added as follows for all the above three units :


C1-90038-S1-01 - Scheduled

C1-90038-S2-01 - Pending

C1-90038-S3-01 - Pending

C1-90038-S4-01 - Pending

C1-90038-S5-01 - Pending

C1-90038-S6-01 - Pending


• Now the employee will go at the client’s address and then he found that AC3 unit is not available on the Address 1.


• At this time, while filling the service report for C1-90038-S1-01, the employee can not select the AC3 unit.


• And he has to add notes like “AC3 unit is not available on the Address1 and change the address of AC3 to Address 2”.


• The Employee has to add the notes. So by this notes, in the listing of Service Reports in the Admin Panel, Admin can find the service reports which has this kind of notes to change the address of the unit.


• In the Service Report listing, the record / row will have the different color than the other rows. So Admin can easily find out this kind of request and can change the address of requested unit without checking notes into all the service reports.


For Scheduled Service : C1-90038-S1-01


• As above, at the time of filling the service report, AC3 unit is not selected by the Employee.


• So for AC3 unit, one new service will be added with “Continuous Previous Work” status.


• Now when the admin will change the address of the unit, the status of added “Continuous Previous Work” service will be changed to the “Address Changed”.


• Or if that service will have any other unit in “Continuous Previous Work” service, then AC3 (for which the address is changed) will be removed from the “Continuous Previous Work” service.


For Pending Services : C1-90038-S2-01, C1-90038-S3-01, C1-90038-S4-01, C1-90038-S5-01, C1-90038-S6-01


• The other above pending services which has the unit AC3, from those all the pending services, AC3 unit will be removed.


• After completing the above process, now for unit AC3, no services will be available in the Admin.


• So now when the scheduler will run, the new 6 services will be added for AC3 units as per the Automatic Rules for the updated address.


• Thus the process of changing the address of added units will be managed.



Case 3: The unit is added and its first service is completed.


• In this case if the client wants to change the unit’s address then in admin panel, while admin going to change the client’s address, admin will prompted with that admin can not change the address because its one or more service is completed.



• In short, before completing any service of the unit, admin can change the client’s address.


• Employee also can add AC unit from employee application on behalf of client using Check, PO or Credit Card.


• To add unit from employee application follow below steps:


               1) Login into application

               2) Open side menu, then select unit list from the menu

               3) Then click on plus button, A Pop-up will be asked same as displayed in the below image.

               4) Then Select any one option and click on OK.




• If ‘Credit Card’ option selected, then Employee can purchase up to five units at the same transaction (same as current client application – limitation driven by stripe processing and system selections), and proceed to purchase.


• The 36 invoices will be added into the database for this specific unit. From 36 invoices, 35 will have the status as “UnPaid” & first invoice will be charged using Stipe service and the status of that invoice will be as “Paid” (assuming the charge went through successfully).



• If “Check’ option selected, then there will be no limit to the number of units that can be added in the same transaction since we are not using stripe and below information will be asked at the time of checkout.



               1) Check Number (optional)

               2) 2 images - Front & back for check (optional)

               3) Notes for Accounting (optional)




• If “PO’ option selected, then there will be no limit to the number of units that can be added in the same transaction since we are not using stripe and below information will be asked at the time of checkout.


                   1) PO Number (Optional)

               2) Check Number (Optional)

               3) Notes for Accounting (Optional)        


               


• After checkout, Based on the Plan duration i.e. 36 entries per unit will be added into the database with 36 invoices per unit with option status “Paid” / “Unpaid”.


• If the check number will be entered from the employee application, then first invoice from 36 will be added as “Paid” and other 35 invoices will be added as “Unpaid”.


• The 36 invoices will be per unit.


• For other 35 invoices, per month admin has to add the check details and mark the invoice as “Paid”.
