Waiting for Approval


•When client purchases a package for the unit, then based on automatic rules, Number of services is scheduled.


•Whenever scheduler will run and if all the automatic rules are followed then the service will be added in this section and client will get a notification to accept or cancel the service.


• In this section, admin can filter the list of waiting for approval service by Service case #, client name, employee name, work area and date range.


• Admin can also delete those services which are requested but still in waiting for approval section. Admin will have check boxes in each row which can be deleted by admin.


•Whenever any service is added in this section, the client will get a notification to accept or cancel the service as shown in the below image(A).


• After clicking on the notification client will redirect to the notification detail page from where he/she can accept or cancel the service as displayed in the below image(B).


• If client Approves the service then the service will be moved from this section to the ‘Scheduled’ section.


• And if client cancels the service then he/she will be asked preferred day, preferred time to reschedule the service and reason for cancellation as displayed in the below Image(C).


• If client cancel the service then service will be displayed as a ‘Cancelled’ in this section. And when the scheduler will run, the service will be scheduled on preferred date and preferred time added by the client at the time of canceling service.


• If client / admin did not accept or reject schedule confirmation within the time specified in Site Settings -> “Minimum Days Before Client can Approve Service”, then that service will automatically move to Pending section with attempt log. Now this service will again schedule when next automatic scheduler run or admin can manually schedule.
